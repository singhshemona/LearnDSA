Selection Sort O(n2)

Description

Selection sort is best used to sort small sequences due to its poor runtime. It is also effective if the sequence is almost sorted, and is used with arrays or lists.

Procedure

Find the smallest element. Swap it with the first element. Find the second-smallest element. Swap it with the second element. Find the third-smallest element. Swap it with the third element. Repeat finding the next-smallest element, and swapping it into the correct position until the array is sorted.

Runtime Explaination

Inserting elements altogether takes O(n) time, the time to remove elements is proportional to the number of elements, therefore it runs in O(n2) time.

Examples

Suppose you had a deck of unsorted cards in your hand and you wanted to sort them, with the smallest elements on your left and the largest elements on your right. You would start by finding the smallest card in the list, putting that card as the first card in your sorted section, finding the next smallest card in the list, putting that card as the second card in your sorted section, and so on until you have a sorted list.
