Longest Common Subsequence (LCS) O(n+m)

Description

Given 2 strings X and Y, the algorithm finds the longest common subsequence (LCS) of both X and Y. a sequence that can be obtained from another sequence by deleting some elements without changing the order of the remaining elements. For example, the LCS of ABCDEFG and XZACKDFWGH is ACDFG.

Procedure

  1. Consider two sequences, X and Y. Let X be of length m = X[0..m-1] and Y be of length n = Y[0..n-1] and Let L(X[0..m-1], Y[0..n-1]) stand for the length of the LCS of X and Y
  2. If the last characters of both sequences match (in other words, X[m-1] == Y[n-1]) then apply this rule: L(X[0..m-1], Y[0..n-1]) = 1 + L(X[0..m-2], Y[0..n-2])
  3. If last characters of both sequences donâ€™t match (in other words, X[m-1] !=Y[n-1]) then apply this rule: L(X[0..m-1], Y[0..n-1]) = MAX ( L(X[0..m-2], Y[0..n-1]), L(X[0..m-1], Y[0..n-2])

Runtime Explaination

The while loop is done n times since we remove one of the nodes each time, so time for while loop is O(n log n). Total time is therefore O(n log n) + O(n log n), or just O(n log n)

You could use LCS to determine how similar two documents are for plagiarism.
